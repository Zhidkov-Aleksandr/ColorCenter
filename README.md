# 🎨 Колеровочный Центр (Tinting Center / Dual System)

Промышленный программный комплекс для расчета рецептур колеровки лакокрасочных материалов (ЛКМ). Программа объединяет две фундаментально разные системы колеровки (**Innovatint** и **Datacolor**) в едином, быстром и интуитивно понятном графическом интерфейсе.

Создано для оптимизации работы колеровщиков, устранения ошибок при пересчете пигментов и автоматизации выдачи заданий на производство.

---

## 🌟 Ключевые возможности

* **Двойная архитектура расчетов:**
  * **Innovatint:** Расчет по объемным долям с автоматическим умножением на плотность (Specific Gravity) каждого пигмента.
  * **Datacolor:** Прямой весовой расчет в граммах на килограмм базы.
* **Умная работа с базами (White vs Clear):**
  * Для *белых (укрывистых)* баз пигменты рассчитываются сверх объема основы.
  * Для *прозрачных* баз реализован **принцип пропорционального замещения**. Для системы Datacolor доступен выбор пропорции **70/30** или **80/20**, где пигменты замещают часть связующего лака для сохранения физико-химических свойств краски.
* **Интеллектуальный поиск цветов:** Поиск игнорирует регистр, автоматически очищает системный "мусор" (GUID-хэши) и сливает дублирующиеся названия, оставляя только чистые стандарты (RAL, NCS, Pantone, Benjamin Moore, Sherwin Williams и др.).
* **Интеграция с производством:**
  * Генерация и моментальная печать текстового «Задания на колеровку» (чека) с датой, пропорциями и клиентом.
  * Кнопка быстрого копирования итогового веса в буфер обмена для вставки в кассовую/учетную программу (1С/ERP).

---

## 📂 Структура проекта

* `gui_tinting.py` — Основное графическое приложение (написано на Python / Tkinter). Не требует установки тяжелых серверов, работает полностью автономно.
* `convert_datacolor.py` — Умный SQL-конвертер. Подключается к промышленной базе MS SQL Server Datacolor, отфильтровывает черновики и локальные правки, выбирает *самые свежие утвержденные стандарты* и упаковывает их в легкий JSON-файл. Сжимает базу с сотен мегабайт до ~50 Кб.
* `innovatint.db` — База данных рецептур системы Innovatint (текстовый формат).
* `datacolor.json` — Скомпилированная и очищенная база данных рецептур Datacolor.

---

## 🛠 Технологии

* **Язык:** Python 3.10+
* **GUI:** Tkinter (с использованием ttk styles для современного плоского дизайна).
* **Базы данных:** JSON (Datacolor), Custom Text/DB (Innovatint), `pyodbc` (для связи с MS SQL Server на этапе конвертации).

## 🚀 Установка и запуск

### Вариант 1: Использование готового .exe (Рекомендуется для производства)
Для работы на рабочих станциях колеровщиков программа скомпилирована в один независимый исполняемый файл с помощью `PyInstaller`.
1. Поместите файл `gui_tinting.exe` в удобную папку.
2. Положите рядом с ним файлы `innovatint.db` и `datacolor.json`.
3. Запустите `.exe` (не требует установки Python на ПК пользователя).

### Вариант 2: Запуск из исходного кода
1. Склонируйте репозиторий:
```bash
   git clone [https://github.com/Zhidkov-Aleksandr/ColorCenter.git](https://github.com/Zhidkov-Aleksandr/ColorCenter.git)
```
   Убедитесь, что установлен Python 3.

Запустите приложение:

Bash
 ```bash
python gui_tinting.py
```

🔄 Обновление базы Datacolor
Если на производстве появились новые рецепты или каталоги, базу можно легко обновить без перекомпиляции самой программы:

Запустите скрипт convert_datacolor.py на компьютере, где установлен MS SQL Server с базой YDatacolorLab.

Скрипт автоматически найдет колонки, отфильтрует старые версии рецептов и создаст свежий файл datacolor.json.

Замените старый datacolor.json в папке с программой на новый. При следующем запуске программа подхватит новые цвета.
